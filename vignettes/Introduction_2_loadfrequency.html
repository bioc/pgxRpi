pgxLoader function in this package

This function loads various data from Progenetix database.

The parameters of this function:

  • type A string specifying output data type. Avaiable options are “biosample”, “variant” or “frequency”.
  • output A string specifying output file format. Avaiable options are NULL, ‘pgxseg’ or ‘seg’ when the parameter type is “variant”; ‘pgxseg’ or ‘pgxmatrix’ when the paramter type is “frequency”.
  • group_id A single or a comma-concatenated list of identifiers for cancer type, literature, and cohorts such as c(“NCIT:C7376”,“icdom-98353”,“PMID:22824167”, “pgxcohort-TCGAcancers”). Only used if the parameter type is “biosample” or “frequency”.
  • codematches A logical value determining whether to exclude samples from child terms of specified group id. If FALSE, retrieved samples include child terms of specified group id. Only used when group_id is specified. Default is FALSE.
  • biosample_id A single or a comma-concatenated list of identifiers used in Progenetix database for identifying biosamples.
  • save_file A logical value determining whether to save the variant data as file instead of direct return. Only used when the paramter type is “variant”. Default is FALSE.
  • filename A string specifying the path and name of the file to be saved. Only used if the parameter save_file is TRUE. Default is “variants.seg/pgxseg” in current work directory.

Retrieve CNV frequency data

Relevant parameters of pgxLoader function: type, output, group_id, codematches

There are two output formats

The first output format (output = “pgxseg”)

freq_pgxseg <- pgxLoader(type="frequency", output ='pgxseg',group_id=c("NCIT:C4038","icdom-85003"), 
                         codematches = TRUE)
#> 
#>  accessing IntervalFrequencies service from Progenetix

The parameter codematches determines whether the calculation of CNV frequency excludes samples from child terms.

The meta slot of this object looks like this

freq_pgxseg$meta
#>          code                            label sample_count
#> 1  NCIT:C4038             Lung Carcinoid Tumor          141
#> 2 icdom-85003 Infiltrating duct carcinoma, NOS        12662

The data slot contains a list of the CNV frequency matrices named by the corresponding group id. The matrix with the name ‘total’ is the the CNV frequency matrix where all group ids are concatenated.

names(freq_pgxseg$data)
#> [1] "NCIT:C4038"  "icdom-85003" "total"

You can access the pgxseg matrix of one id from this list.

head(freq_pgxseg$data[['NCIT:C4038']])
#>     group_id reference_name   start     end gain_frequency loss_frequency index
#> 1 NCIT:C4038              1       0 1000000          0.709          8.511     0
#> 2 NCIT:C4038              1 1000000 2000000          2.128          8.511     1
#> 3 NCIT:C4038              1 2000000 3000000          2.128         12.057     2
#> 4 NCIT:C4038              1 3000000 4000000          2.837         12.766     3
#> 5 NCIT:C4038              1 4000000 5000000          2.837          9.929     4
#> 6 NCIT:C4038              1 5000000 6000000          2.837          8.511     5

In the pgxseg matrix,

  • group_id: identifiers for grouping.
  • chro, start and end represent interval location.
  • gain_frequency and loss_frequency indicate the percent values for gains and losses overlapping the segment, respectively

The second output format (output = “pgxmatrix”)

This function can select id of interests from input object and return a new object with selected ids.

Visualization of CNV frequency data using ‘.pgxseg’ file

pgxFeqplot function

This function provides CNV frequency plots by genome or chromosomes as you request.

The parameters of this function:

  • data: frequency data returned by pgxLoader function. The frequency matrices in data slot must stored as .pgxseg format, which can be specified by output parameter of pgxLoader function.
  • chrom: a vector with chromosomes to be plotted. If NULL, return the plot by genome. If specified the frequencies are plotted with one panel for each chromosome. Default is NULL.
  • layout: number of columns and rows in plot. Only used in plot by chromosome. Default is c(1,1).
  • group_id: a string value to indicate which group id to be plotted. The length of id is limited to one if the parameter circos is False. Default is the first group id.
  • circos: a logical value to indicate if return a circos plot. If TRUE, it can return a circos plot with multiple group ids for display and comparison. Default is FALSE.

CNV frequency plot by genome (GRCh38)

pgxFreqplot(freq_pgxseg, group_id='icdom-85003')

CNV frequency plot by chromosomes

pgxFreqplot(freq_pgxseg, chrom=c(1,2,3), layout = c(3,1), group_id='NCIT:C4038')

CNV frequency circos plot

pgxFreqplot(freq_pgxseg, circos = TRUE,group_id = 'icdom-85003')

The circos plot also supports multiple group comparison

pgxFreqplot(freq_pgxseg,circos = TRUE,group_id= c("NCIT:C4038","icdom-85003"))

Clustering of CNV frequency data using ‘.pgxmatrix’ file

Hierarchical clustring

Use hierarchical clustring to cluster the tumor types which are picked before by NCIt code.

freq_mat <- freq_pgxmatrix$data[['total']][,2:dim(freq_pgxmatrix$data[['total']])[2]]
rownames(freq_mat) <- freq_pgxmatrix$data[['total']]$group_id
dist_mat <- dist(freq_mat, method = 'euclidean')
hclust_res <- hclust(dist_mat, method = "ward.D")

The cluster result is showed below.

hcd <- as.dendrogram(hclust_res)
plot(hcd, type = "rectangle", ylab = "Height")
# Based on the plot, the number of clusters is set to 3.
cut<- cutree(hclust_res, k = 4)
rect.hclust(hclust_res , k = 4, border = 2:6)

Merge frequency matrix

Merge the frequency matrix by calculating the difference between gain frequency and loss frequency per interval (gain-loss)

diff_mat <- freq_mat[,1:3102]-freq_mat[,3103, 6204]

Sort the merged matrix by clustring membership

sort_idx <- order(cut)
cut <- cut[sort_idx]
diff_mat <- diff_mat[sort_idx,]

Get metadata (label, sample number)

From the meta slot of the object

meta <- freq_pgxmatrix$meta[sort_idx,]

Plot heatmap

Plot a heatmap to show the merged matrix. The color in rowside shows cluster membership calculated by the original frequency matrix

library("RColorBrewer")
library('pheatmap')
rownames(diff_mat) <- paste(meta[,1], paste0("(",meta[,3],")"),meta[,2])

col <- colorRampPalette(brewer.pal(10, "RdYlBu"))(256)
col_2 <- brewer.pal(n = 8, name = 'Dark2')
parlength <- 100
myBreaks <- c(seq(min(diff_mat), 0, length.out=ceiling(parlength/2) + 1), 
              seq(max(diff_mat)/parlength, max(diff_mat), length.out=floor(parlength/2)))

annotation_row <- data.frame(cluster=factor(cut))
annotation_col <- data.frame(chr  = factor(substring(colnames(diff_mat),1,
                                                    last=regexpr("\\.", colnames(diff_mat))-1),
                                          levels = c(gsub('^','X',seq(1,22)), 'X','Y')))
rownames(annotation_col) <- colnames(diff_mat)
rownames(annotation_row) <- rownames(diff_mat)

pheatmap(as.matrix(diff_mat), scale = "none",cluster_cols = FALSE, cluster_rows = FALSE,width = 8,height = 10,cellwidth=0.1,breaks=myBreaks,show_colnames=FALSE ,
         annotation_row = annotation_row,annotation_col = annotation_col,annotation_legend=TRUE)
